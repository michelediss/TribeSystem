# TribeBox Trigger / Pitch Engine

Status: **defined**

TribeBox must not confuse the rhythm generator with the note generator.

The Sequencer Engine is therefore split into two independent musical layers:

```text
Trigger Engine = when a sound is triggered
Pitch Engine   = which note is played
```

This separation allows TribeBox to combine simple drum-machine behavior, Euclidean rhythm generation, manual note entry, root-aware tuning, scale correction, and controlled generative pitch behavior without making the sequencer unpredictable.

Example:

```text
Track 7 Acid
Trigger Engine = EUCLIDEAN
Pitch Engine   = GENERATIVE
Root           = E
Scale          = phrygian
```

Meaning:

```text
Rhythm is generated by the Euclidean Trigger Engine.
Notes are generated by the Pitch Engine inside the selected musical context.
```

---

## Architectural Rule

Trigger decisions and pitch decisions are separate.

A step or track may answer two different questions:

```text
Should this step play?
Which pitch should it use?
```

The first question belongs to the Trigger Engine.

The second question belongs to the Pitch Engine.

The Trigger Engine must never silently generate musical note choices. The Pitch Engine must never silently add rhythmic triggers outside the trigger layer.

---

## Project-Level Musical Context

A Project or Pattern defines the shared musical context used by tonal tracks:

```text
root_note
scale
```

Initial scale set:

```text
chromatic
major
natural_minor
minor_pentatonic
dorian
phrygian
```

Default:

```text
root_note: C
scale: chromatic
```

The root/scale context is used by Root Lock, Scale Lock, and Generative pitch behavior.

Atonal tracks may ignore this context.

---

## Track-Level Engine Model

Each track stores both a Trigger Engine configuration and a Pitch Engine configuration.

```text
Track
├─ trigger_engine
│  ├─ mode
│  └─ mode_parameters
└─ pitch_engine
   ├─ mode
   └─ mode_parameters
```

The two engines are evaluated independently, then resolved together during playback.

Example combinations:

```text
Kick: Step + Root Lock
Bass: Euclidean + Generative
Acid: Step + Scale Lock
Perc: Euclidean + Off
Stab: Step + Root Lock
```

---

## Trigger Engine

The Trigger Engine decides when a sound is activated.

Initial modes:

```text
STEP
EUCLIDEAN
```

### STEP

Classic drum-machine step sequencing.

Example:

```text
X--- X--- X--- X---
```

This is the default sequencing model and remains the most direct mode for kicks, snares, hats, percussion, basslines, acid lines, stabs, and sample triggering.

STEP mode uses the visible step grid directly:

```text
step.active = true / false
```

### EUCLIDEAN

Euclidean mode distributes a number of hits across a given length.

Parameters:

```text
steps
hits
rotation
```

Example:

```text
steps: 16
hits: 5
rotation: 2
```

This produces more organic patterns and is useful for:

- percussion
- hats
- noise hits
- broken basslines
- acid patterns
- stabs
- utility sampler phrases

Euclidean mode generates trigger placement only. It does not choose musical notes.

---

## Pitch Engine

The Pitch Engine decides which pitch or musical note is used when a trigger occurs.

Initial modes:

```text
OFF
MANUAL
ROOT_LOCK
SCALE_LOCK
GENERATIVE
```

Not every mode is useful on every track.

---

## Pitch Mode: OFF

OFF is used for non-tonal sounds.

Examples:

- hat
- clap
- noise
- crash
- ride
- shaker
- atonal percussion
- noise FX

In OFF mode, TribeBox applies no musical note logic.

The track may still expose free pitch/tune controls for sound design, but these are not interpreted as scale-aware musical notes.

---

## Pitch Mode: MANUAL

MANUAL means the user writes the note values explicitly.

Example:

```text
Step 01 = E
Step 05 = G
Step 09 = B
Step 13 = D
```

This is useful for:

- hand-written basslines
- acid lines
- lead lines
- stabs
- tonal sample triggering
- deliberate chromatic movement

Manual notes may be chromatic unless combined with Scale Lock behavior.

---

## Pitch Mode: ROOT_LOCK

ROOT_LOCK anchors a sound to the project or pattern root note.

Example:

```text
Project Root = E
Kick Root    = E
Bass Root    = E
Acid Root    = E
```

Root Lock prevents tonal elements from drifting away from the main key center.

It is most useful on:

- Kick
- Bass / Rumble
- Acid / Lead
- tonal samples
- stabs
- vocal chops
- synth hits

### Kick Behavior

Kick tuning is special.

A kick may have a root note, but it should not be continuously transposed like a melody because excessive pitch shifting changes the punch, body, and transient behavior.

For Kick, Root Lock is primarily a selection and tuning reference:

```text
Project Root = E
Kick Root    = E or compatible interval
```

The goal is coherence with bass and rumble, not melodic pitch generation.

---

## Pitch Mode: SCALE_LOCK

SCALE_LOCK constrains entered or generated notes to the selected root and scale.

Example:

```text
root_note: E
scale: phrygian
```

If a note outside the scale is entered or produced, TribeBox resolves it to the nearest valid note in the scale.

Scale Lock is useful for:

- Bass
- Acid
- Lead
- Stab
- tonal vocal chops
- tonal synth hits
- melodic sampler material

Scale Lock is not useful for:

- hats
- claps
- snares
- noise
- atonal percussion
- non-tonal FX

Rule:

```text
Scale Lock corrects notes.
It does not create notes by itself.
```

---

## Pitch Mode: GENERATIVE

GENERATIVE is the actual note generator.

In this mode, TribeBox chooses notes automatically within musical rules defined by the user.

Initial parameters:

```text
root_note
scale
range
density
probability
evolution
octave_spread
randomness
```

Example:

```text
root_note: E
scale: phrygian
range: 1_octave
density: medium
evolution: 30%
octave_spread: 1
randomness: 20%
```

Expected result:

```text
The line changes over time or through explicit generation commands,
but remains musically coherent with the selected root and scale.
```

For v0.1, generative behavior should remain controllable and recallable. The engine should prefer explicit generation/mutation actions over invisible uncontrolled playback changes unless a future live-generative mode is deliberately specified.

---

## Scale Lock vs Generative vs Root Lock

These modes must remain conceptually separate.

| Mode | Meaning |
| --- | --- |
| OFF | No tonal logic |
| MANUAL | User chooses the notes |
| ROOT_LOCK | Sound is anchored to the project/pattern root |
| SCALE_LOCK | Notes are corrected into the selected scale |
| GENERATIVE | TribeBox creates note choices inside musical rules |

Practical distinction:

```text
Scale Lock:
I enter notes; TribeBox corrects them.

Generative:
TribeBox proposes notes.

Root Lock:
TribeBox keeps the sound coherent with the project root.
```

## Per-Track Application

### Track 1 — Kick

Recommended Pitch Engine modes:

```text
ROOT_LOCK
MANUAL
```

Not recommended:

```text
SCALE_LOCK
GENERATIVE
```

The kick should be selected or tuned relative to the root, but not generated melodically.

### Track 2 — Bass / Rumble

Recommended Pitch Engine modes:

```text
ROOT_LOCK
SCALE_LOCK
GENERATIVE
MANUAL
```

Bass is one of the main tonal anchors of TribeBox.

The Note Generator makes sense here when constrained by root, scale, range, and density.

### Track 3 — Perc 1

Recommended Pitch Engine modes:

```text
OFF
MANUAL
```

Usually no scale logic. Manual pitch remains useful for sound design.

### Track 4 — Perc 2

Recommended Pitch Engine modes:

```text
OFF
MANUAL
```

Usually no scale logic. Manual pitch remains useful for sound design.

### Track 5 — Hat / Ride

Recommended Pitch Engine modes:

```text
OFF
MANUAL
```

Usually no scale logic. Manual pitch remains useful for tuning or texture.

### Track 6 — Perc / Noise / Utility

Recommended Pitch Engine modes:

```text
OFF
MANUAL
```

Future noise or synth extensions may add tonal behavior, but the default assumption is atonal.

### Track 7 — Acid / Lead

Recommended Pitch Engine modes:

```text
MANUAL
SCALE_LOCK
GENERATIVE
ROOT_LOCK
```

This is the best track for:

- manual acid lines
- generative acid lines
- Scale Lock
- Root Lock
- slide
- accent

### Track 8 — Utility / Sampler

Track 8 depends on the loaded sample.

```text
atonal sample   → OFF
free pitch use  → MANUAL
tonal sample    → ROOT_LOCK / SCALE_LOCK
melodic sample  → GENERATIVE optional
```

Examples:

```text
Vocal chop → Root Lock useful
Stab       → Scale Lock useful
Noise FX   → Off
```

## Sample Metadata

Root Lock and Scale Lock on samples require metadata.

Each tonal sample should be able to declare its musical identity.

Example sidecar metadata:

```json
{
  "file": "stab_01.wav",
  "type": "tonal",
  "root_note": "E"
}
```

If the Project root is G and the sample root is E, TribeBox can calculate the required transposition:

```text
E → G = +3 semitones
```

Minimum useful metadata for tonal sample behavior:

```text
sample_type: tonal / atonal / loop / one_shot
root_note
scale, optional
bpm, optional for loops
slice_map, optional for sliced material
```

Atonal samples do not require root metadata.

## Playback Resolution

At playback time, TribeBox resolves trigger and pitch behavior in a predictable order.

Suggested order:

```text
1. Read track Trigger Engine mode
2. Determine whether the current step produces a trigger
3. Apply trig conditions and probability
4. If trigger passes, read Pitch Engine mode
5. Resolve note/pitch according to Pitch Engine
6. Apply step note locks, sample locks, slice locks, and parameter locks
7. Apply accent, slide, retrig, microtiming, and performance modifiers
8. Send final event to the sound engine
```

Important rule:

```text
No trigger = no pitch event.
```

Pitch Engine output only matters when the Trigger Engine produces an active event.

```json

  "note": "E2",
  "accent": true,
  "slide": false,
  "probability": 100,
  "locks": {}
```

## UI Implications

The UI should present Trigger Engine and Pitch Engine as two separate concepts.

Track-level page example:

```text
TRIGGER
Mode: Step / Euclidean

PITCH
Mode: Off / Manual / Root Lock / Scale Lock / Generative
Root: Project / custom
Scale: Project / custom
```

This prevents users from thinking that Euclidean rhythm generation also generates notes, or that Generative pitch behavior changes the rhythm.

---

## Final Decision

TribeBox uses a two-layer sequencing architecture:

```text
Trigger Engine = when a sound plays
Pitch Engine   = which note or pitch is used
```

Each track has an independent Trigger Engine and Pitch Engine.

Initial Trigger Engine modes:

```text
STEP
EUCLIDEAN
```

Initial Pitch Engine modes:

```text
OFF
MANUAL
ROOT_LOCK
SCALE_LOCK
GENERATIVE
```

Root Note and Scale belong to the shared musical context and are used by tonal tracks, Root Lock, Scale Lock, Generative pitch behavior, and tonal sample metadata.

The Note Generator is not a classic arpeggiator. It is a controlled generative Pitch Engine constrained by root, scale, range, density, probability, evolution, octave spread, and randomness.
