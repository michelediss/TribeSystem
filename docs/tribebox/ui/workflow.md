# Workflow

This document describes the intended TribeBox interaction workflow.

## Design Goals

- Fast live operation
- Minimal menu navigation
- Physical controls first
- Screen used for feedback and editing

## Main Workflow

1. Select a track.
2. Edit steps or parameters.
3. Use encoders for sound shaping.
4. Use performance buttons during live play.
5. Save project or pattern changes.

## Live Performance Actions

Common actions should be available without deep menus:

- Mute track
- Select track
- Trigger fill
- Apply retrigger
- Change pattern
- Edit macro values

## Editing Workflow

Detailed editing can use the display:

- Sample selection
- Project settings
- Scale and root note settings
- Pattern management
- System settings

## Principle

The performer should be able to play most of the system without relying only on the touchscreen.
